BUGS FIXED IN THIS SESSION
=====================================================================

1. TCP SOCKET CORRUPTION AFTER RAPID WRITES — FIXED
   Root cause: _recv_buffer not cleared on reconnect, causing response
   desync. After ~15 writes, responses from previous commands leaked
   into the buffer for the next command.
   Fix: Clear _recv_buffer on connect(), added auto-retry with fresh
   socket in send_command (max 2 attempts), removed unnecessary delays.

2. M4L BRIDGE PORT NOT RECLAIMED AFTER SERVER RESTART — FIXED
   Root cause: get_m4l_connection failed permanently after one bad ping.
   Fix: Added ping check on existing connections, stale UDP data
   draining before sends, auto-retry in M4L send_command, graceful
   reconnection.

3. CLIP NOTE OPERATIONS FAILING ON LIVE 12 — FIXED
   Root cause: Remote Script used deprecated Live 10 API (remove_notes)
   which silently fails in Live 12.
   Fix: Updated _clear_clip_notes, _quantize_clip_notes, and
   _transpose_clip_notes to use remove_notes_extended (Live 11+ API)
   with fallback to legacy remove_notes for older versions.
   _quantize_clip_notes also now uses Live's built-in clip.quantize()
   for standard grid sizes.

4. GROUP TRACK CRASH ON get_track_info — FIXED
   Root cause: Group tracks don't support track.arm / track.can_be_armed
   and other properties. Accessing them threw AttributeError, crashing
   the entire function.
   Fix: Wrapped all track-type-specific properties (arm, is_foldable,
   has_audio_input, has_midi_input) in try/except blocks. Added
   is_group_track field to the response.

5. MCP SERVER RUNNING FROM STALE .whl PACKAGE — FIXED
   Root cause: Claude Desktop config pointed to a prebuilt .whl in dist/,
   not the source code. All code edits to server.py had no effect until
   the package was rebuilt with `uv build`.
   Fix: Rebuilt the .whl package. Also improved error messages in
   server.py to return actual exception text instead of generic messages.

6. RAPID READS STRESS TEST — PASSED
   9 parallel get_track_info calls (including group track) all returned
   correctly with no response mixing or corruption.
